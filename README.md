# Claude Watch

[English](README.md) · [中文](README.zh-CN.md)

A macOS menu bar app that watches your `claude.ai` 5-hour and 7-day usage windows. Sits in the top-right of your screen, polls every 90 seconds, and tells you at a glance how close you are to running out.

## Install

1. Download `ClaudeWatch-X.Y.Z.dmg` from the [latest release](https://github.com/metalbreeze/claudewatch/releases).
2. Open the DMG, drag **Claude Watch** to Applications.
3. **First launch**: the app is ad-hoc signed (no Apple notarization yet), so macOS will block it with *"Apple could not verify…"*. Workaround:
   - In Applications, **right-click** Claude Watch → **Open** → confirm in the dialog.
   - macOS remembers this; double-click works thereafter.
4. The menu bar icon appears as `⌬ ⏳`. Right-click it → **Import from cURL…**.
5. Paste a cURL from your browser DevTools (see below). Done.

Requirements: Apple Silicon Mac, macOS 13+. Intel Macs and notarized signing are on the roadmap.

## Why a cURL paste, not a login form?

Claude.ai's web UI is protected by Cloudflare's bot detection and Google's OAuth restrictions on embedded WebViews. Embedding a login window inside the app fights both layers and produces a poor login experience (captcha loops, "this browser may not be secure" rejections from Google SSO).

The pragmatic solution: you sign in to claude.ai in your real browser anyway, so we just borrow the resulting session. Open DevTools → Network tab → reload the usage page → right-click the request to `/api/organizations/{your-org-uuid}/usage` → **Copy as cURL** → paste into the app.

The app extracts the `sessionKey` cookie, the `cf_clearance` cookie, and the User-Agent string from your cURL, stores them locally, and uses them to poll the same endpoint every 90 seconds.

## Where your data lives

- **Cookies & device ID**: `~/Library/Application Support/ClaudeWatch/secrets/`, file permissions `0600`. Never transmitted anywhere except `claude.ai` itself.
- **Usage history**: `~/Library/Application Support/ClaudeWatch/usage.db` (SQLite). Local only.
- **No analytics, no telemetry, no third-party services.** The only network destination is `claude.ai`.

When cookies expire (~24 hours for `cf_clearance`, weeks for `sessionKey`), you do another 30-second cURL paste to refresh.

## What's in the popover

Clicking the menu bar icon opens a 340pt-wide popover:

- **Two gauge cards** at the top: `5H` (green tint) and `Week` (teal tint). Big percentage number turns yellow at ≥75%, red at ≥90%.
- **Timeframe picker**: 1h / 8h / 24h / 1w. Each shows a different chart range.
- **Chart** with up to four kinds of lines:
  - **Green solid** — actual 5h-window utilization (every timeframe)
  - **Teal solid** — actual 7-day weekly utilization (1w view only)
  - **Gray dashed** — short-term 5h forecast (1h/8h/24h). Dash gap encodes confidence: `[5,2]` near-solid = high confidence projected hit; `[2,6]` sparse dotted = stable / won't hit.
  - **Indigo dashed verticals** — 5h or weekly window reset boundaries
- **Forecast caption**: "likely full at HH:MM", "won't hit limit this window", or "stable", depending on slope and projection.

## Build from source

```bash
git clone https://github.com/metalbreeze/claudewatch.git
cd claudewatch
brew install xcodegen
xcodegen generate          # produces ClaudeWatch.xcodeproj
open ClaudeWatch.xcodeproj
# In Xcode: select the ClaudeWatchMac scheme, ⌘R
```

To run the package tests (41 tests covering models, storage, scraping, forecasting, alerts, sync):

```bash
cd Packages/UsageCore
swift test
```

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml`. The `.xcodeproj` itself is gitignored.

## Architecture

Single Xcode project with two targets and one local Swift Package:

- **`Packages/UsageCore/`** — pure-logic Swift Package. Models, GRDB-backed SQLite storage, Anthropic API scraper, forecaster, polling controller, notification engine, CloudKit sync. 38+ unit tests.
- **`Apps/ClaudeWatchMac/`** — SwiftUI menu bar app. `LSUIElement = YES` (no Dock icon). Owns the popover, settings window, and the cURL import flow.
- **`Apps/ClaudeWatchiOS/`** — placeholder for the iOS companion app (planned).

Tech stack: Swift 5.10+, SwiftUI, [GRDB.swift](https://github.com/groue/GRDB.swift) for SQLite, Swift Charts, WidgetKit (planned), WKWebView (login fallback only), CloudKit (planned), UserNotifications.

## Caveats and risks

- **Anthropic's `/usage` API is undocumented.** The response shape was reverse-engineered from a real claude.ai session on 2026-04-30. If Anthropic changes the format, the app will surface a `schemaDrift` error in the popover until a new version is shipped.
- **Cookies expire.** `cf_clearance` typically lasts ~24 hours; `sessionKey` lasts longer (days to weeks). When either expires, the popover shows a banner and you do a fresh cURL import.
- **ToS gray area.** Programmatic polling of consumer claude.ai surfaces is technically against Anthropic's Terms of Service, even at 90-second cadence. Account suspension is the realistic worst case (not legal action). Use at your own risk; this is a personal monitoring tool, not a production system.
